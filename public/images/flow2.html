Ingress-NGINX Flow

  • User hits DNS.
  • DNS resolves to the public entry.
  • Load Balancer forwards traffic to Ingress-NGINX.
  • Ingress-NGINX terminates TLS.
  • Ingress rules pick the Service.
  • Service load-balances to Pods.
  • Pods serve the request.

Gateway API Flow

  • User hits DNS.
  • DNS resolves to the public entry.
  • Load Balancer forwards to NGF gateway.
  • Gateway API (nginx-gateway) terminates TLS.
  • HTTPRoutes decide the Service.
  • Service fans out to Pods.
  • Pods serve the request.
